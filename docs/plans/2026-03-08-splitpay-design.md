# SplitPay — Дизайн

> Inline Telegram-бот для разделения расходов между друзьями.

## Проблема

Группа друзей идёт в ресторан. Один платит за всех. Потом начинается:
"кто кому сколько должен?", "я тебе уже переводил", "скинь номер карты".

Splitwise решает это, но требует отдельное приложение. Русскоязычного
Telegram-бота для этого **нет**.

## Решение

Telegram-бот, который работает **inline** в любом чате:

```
@SplitPayBot 3000 за ужин
```

Бот создаёт карточку с суммой, делит на участников, показывает реквизиты
для перевода. Каждый отмечает "Я отдал" — карточка обновляется.

## Целевая аудитория

- Группы друзей (рестораны, поездки, совместные покупки)
- Пары / соседи (аренда, продукты, коммуналка)
- Любые физлица, которым нужно разделить расход

## Пользовательские сценарии

```

   │        за ужин              │
   │                             │
   │  Заплатил: @petya           │
   │  Сбер: +7 999 123 45 67    │
   │                             │
   │  Должны (по 750 ₽):        │
   │  ○ @vasya                   │
   │  ○ @masha                   │
   │  ○ @kolya                   │
   │                             │
   │  [Я отдал ✓]               │
   └─────────────────────────────┘

4. Вася переводит 750₽ по СБП и нажимает "Я отдал ✓"
5. Карточка обновляется:
   │  ✓ @vasya — отдал           │
   │  ○ @masha                   │

```

```
5. Бот: "Готово! Теперь в любом чате: @SplitPayBot 500 за кофе"
```

## Архитектура

### Стек

- Python 3.12, aiogram 3, SQLAlchemy 2.0 async, aiosqlite
- Pillow для рендера карточек
- pydantic-settings для конфигурации
- pytest + pytest-asyncio для тестов
- ruff для линтинга

### Слои

```
Telegram
  ↓
bot/routers/        → обработка команд и callback-ов
  ↓
bot/services/       → бизнес-логика (split, рендер)
  ↓
bot/db/             → модели и репозитории
  ↓
SQLite
```

Правило: routers не обращаются к БД напрямую. Всё через services → repositories.

### Модели данных

```python
class User:
    telegram_id: int        # PK
    username: str | None
    first_name: str
    phone: str | None       # +7... для переводов
    bank_name: str | None   # sber, tinkoff, alfa...
    is_onboarded: bool
    created_at: datetime

class Expense:
    id: int                 # PK, autoincrement
    creator_id: int         # FK → User (кто заплатил)
    amount: int             # сумма в копейках
    description: str        # "за ужин"
    inline_message_id: str | None
    card_file_id: str | None
    created_at: datetime
    participants: list[ExpenseParticipant]

class ExpenseParticipant:
    expense_id: int         # FK → Expense, PK
    user_id: int            # FK → User, PK
    amount: int             # доля в копейках
    is_settled: bool        # отдал или нет
    settled_at: datetime | None
```

### Отличия от TGpay

| Компонент | TGpay (старый) | SplitPay (новый) |
|-----------|---------------|-----------------|
| Модель Payment | creator + участники-подтвердившие | creator + должники с долями |
| Бизнес-логика | "я оплатил" (trust) | split суммы + "я отдал" (trust) |
| Карточка | QR + банк + сумма | сумма + реквизиты + список должников |
| SBP/QR | Генерация ссылок | Убрано |
| Онбординг | Телефон + банк | Телефон + банк (переиспользуем) |
| Inline | Создание платежа | Создание расхода с split |

## Рендер карточки

PNG, генерируется Pillow. Минималистичный светлый дизайн (как текущий).

Элементы:
- Сумма (крупно)
- Описание
- Реквизиты создателя (банк + телефон)
- Список участников с статусом (○ должен / ✓ отдал)

## Определение участников

### В групповом чате (inline)

Бот **не может** получить список участников группы через Telegram API.
Участники определяются через callback: первые N человек, которые нажали
кнопку "Я должен" или "Я отдал", добавляются как участники.

Альтернативный вариант: создатель указывает количество участников:
`@SplitPayBot 3000 за ужин 4` → делит на 4.

### В личных сообщениях (inline)

Один получатель — тот, кому отправлена карточка.

## Что удаляем

- `bot/services/sbp.py` — генерация СБП-ссылок
- `bot/services/payment.py` — старая логика платежей
- qrcode зависимость
- Всё, что связано с QR-кодами

## MVP фичи

1. **Онбординг** — /start → телефон → банк
2. **Inline split** — `@bot 3000 за ужин` → карточка в чате
3. **Кнопка "Я отдал ✓"** — отмечает погашение, обновляет карточку
4. **Реквизиты на карточке** — банк + телефон создателя
5. **Работает в группах и ЛС** — одинаковый inline, разный контекст

## Не входит в MVP

- История расходов / баланс долгов
- Неравный split
- Категории трат
- Статистика
- Напоминания
- Экспорт данных
