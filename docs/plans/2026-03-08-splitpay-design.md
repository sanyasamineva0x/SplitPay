# SplitPay — Дизайн

> Inline Telegram-бот для разделения расходов между друзьями.

## Проблема

Группа друзей идёт в ресторан. Один платит за всех. Потом начинается:
"кто кому сколько должен?", "я тебе уже переводил", "скинь номер карты".

Splitwise решает это, но требует отдельное приложение. Русскоязычного
Telegram-бота для этого **нет**.

## Решение

Telegram-бот, который работает **inline** в любом чате:

```
@SplitPayBot 3000 за ужин
```

Бот создаёт карточку с суммой и реквизитами. Участники чата нажимают
"Я должен" — бот пересчитывает доли. После перевода отмечают "Я отдал" —
карточка обновляется.

## Целевая аудитория

- Группы друзей (рестораны, поездки, совместные покупки)
- Пары / соседи (аренда, продукты, коммуналка)
- Любые физлица, которым нужно разделить расход

## Пользовательские сценарии

### Сценарий 1: Групповой чат

```
1. Петя оплатил ужин за четверых
2. В групповом чате: @SplitPayBot 3000 за ужин
3. Появляется карточка (пока без участников):

   ┌─────────────────────────────┐
   │        3 000 ₽              │
   │        за ужин              │
   │                             │
   │  Заплатил: @petya           │
   │  Сбер: +7 999 123 45 67    │
   │                             │
   │  Нажмите "Я должен",       │
   │  чтобы разделить счёт      │
   │                             │
   │  [Я должен 💰] [Я отдал ✓] │
   └─────────────────────────────┘

4. Вася, Маша и Коля нажимают "Я должен 💰"
   Карточка обновляется с каждым нажатием:

   │  Должны (по 1 000 ₽):      │
   │  ○ @vasya                   │
   │  ○ @masha                   │
   │  ○ @kolya                   │

   (при 2 участниках было по 1500₽, при 3 — по 1000₽)

5. Вася переводит 1000₽ по СБП и нажимает "Я отдал ✓"
   Карточка обновляется:

   │  ✓ @vasya — отдал           │
   │  ○ @masha                   │
   │  ○ @kolya                   │
```

### Сценарий 2: Личные сообщения

```
1. Петя отправляет Васе в ЛС: @SplitPayBot 750 за ужин
2. Появляется карточка:

   ┌─────────────────────────────┐
   │        750 ₽                │
   │        за ужин              │
   │                             │
   │  Перевести: @petya          │
   │  Сбер: +7 999 123 45 67    │
   │                             │
   │  [Я отдал ✓]               │
   └─────────────────────────────┘

3. Вася переводит и нажимает "Я отдал ✓"
   Карточка показывает: ✓ Оплачено
```

В ЛС кнопка "Я должен" не нужна — получатель один.
Бот не может определить кому отправлено сообщение (ограничение
Telegram inline API), поэтому "Я отдал" может нажать любой,
кто видит карточку.

### Сценарий 3: Онбординг

```
1. /start
2. Бот: "Привет! Введи номер телефона, чтобы друзья знали куда переводить"
3. Пользователь: "+7 999 123 45 67"
4. Бот: "Выбери банк" → [Сбер] [Т-Банк] [Альфа] [ВТБ] [Райффайзен] [Другой]
5. Бот: "Готово! Теперь в любом чате: @SplitPayBot 500 за кофе"
```

## Архитектура

### Стек

- Python 3.12, aiogram 3, SQLAlchemy 2.0 async, aiosqlite
- Pillow для рендера карточек
- pydantic-settings для конфигурации
- pytest + pytest-asyncio для тестов
- ruff для линтинга

### Слои

```
Telegram
  ↓
bot/routers/        → обработка команд и callback-ов
  ↓
bot/services/       → бизнес-логика (split, рендер)
  ↓
bot/db/             → модели и репозитории
  ↓
SQLite
```

Правило: routers не обращаются к БД напрямую. Всё через services → repositories.

### Модели данных

```python
class User:
    telegram_id: int        # PK
    username: str | None
    first_name: str
    phone: str | None       # +7... для переводов
    bank_name: str | None   # sber, tinkoff, alfa...
    is_onboarded: bool
    created_at: datetime

class Expense:
    id: int                 # PK, autoincrement
    creator_id: int         # FK → User (кто заплатил)
    amount: int             # сумма в копейках (100 = 1₽)
    description: str        # "за ужин"
    inline_message_id: str | None
    card_file_id: str | None
    created_at: datetime
    participants: list[ExpenseParticipant]

class ExpenseParticipant:
    expense_id: int         # FK → Expense, PK
    user_id: int            # FK → User, PK
    amount: int             # доля в копейках
    is_settled: bool        # отдал или нет
    settled_at: datetime | None
```

### Отличия от TGpay

| Компонент | TGpay (старый) | SplitPay (новый) |
|-----------|---------------|-----------------|
| Модель Payment | creator + участники-подтвердившие | creator + должники с долями |
| Бизнес-логика | "я оплатил" (trust) | split суммы + "я отдал" (trust) |
| Карточка | QR + банк + сумма | сумма + реквизиты + список должников |
| SBP/QR | Генерация ссылок | Убрано |
| Онбординг | Телефон + банк | Телефон + банк (переиспользуем) |
| Inline | Создание платежа | Создание расхода с split |

## Рендер карточки

PNG, генерируется Pillow. Минималистичный светлый дизайн (как текущий).

Элементы:
- Сумма (крупно)
- Описание
- Реквизиты создателя (банк + телефон)
- Список участников с статусом (○ должен / ✓ отдал) и суммой доли
- Если нет участников: "Нажмите «Я должен», чтобы разделить счёт"

## Определение участников

### В групповом чате (inline)

Бот **не может** получить список участников группы через Telegram API.
Участники добавляются динамически: каждый, кто нажал "Я должен",
становится участником. При каждом новом участнике доли **пересчитываются**
поровну.

Пример: расход 3000₽
- 1 участник нажал → 3000₽
- 2 участника → по 1500₽
- 3 участника → по 1000₽

### В личных сообщениях (inline)

Один получатель. Кнопка "Я должен" не показывается — только "Я отдал ✓".
Сумма = полная сумма расхода (пользователь сам указывает нужную).

## Валидация ввода

- **Минимальная сумма**: 1₽ (100 копеек)
- **Максимальная сумма**: 1 000 000₽ (100 000 000 копеек)
- **Описание**: обязательно, 1-100 символов
- **Парсинг**: `@SplitPayBot <сумма> <описание>`
  - Сумма — целое число или с точкой/запятой (500, 500.50, 500,50)
  - Описание — всё после суммы
  - Если не распознано — бот показывает подсказку

## Деление с остатком

Сумма в копейках делится нацело. Остаток добавляется первому участнику.

Пример: 1000₽ (100000 копеек) / 3 = 33333 копейки + 1 остаток.
- Участник 1: 333₽ 34 коп
- Участник 2: 333₽ 33 коп
- Участник 3: 333₽ 33 коп

## Миграция с TGpay

Старая БД (tgpay.db) с таблицами Payment/PaymentParticipant **удаляется**.
Новая БД (splitpay.db) создаётся с нуля при первом запуске.
Пользователям нужно пройти онбординг заново.

## MVP фичи

1. **Онбординг** — /start → телефон → банк
2. **Inline split** — `@bot 3000 за ужин` → карточка в чате
3. **Кнопка "Я должен 💰"** — добавляет участника, пересчитывает доли
4. **Кнопка "Я отдал ✓"** — отмечает погашение, обновляет карточку
5. **Реквизиты на карточке** — банк + телефон создателя
6. **Работает в группах и ЛС** — одинаковый inline, разный контекст

## Не входит в MVP

- История расходов / баланс долгов
- Неравный split
- Категории трат
- Статистика
- Напоминания
- Экспорт данных
